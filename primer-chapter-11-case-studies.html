Day 3-5: OAuth Implementation

Claude Code implemented the OAuth flow with careful attention to security:

// Claude Code generated this complete OAuth implementation
export class StripeConnectService {
  private readonly stripe: Stripe;
  private readonly oauth: OAuthService;
  
  async initiateConnect(userId: string): Promise<string> {
    // Generate state with CSRF protection
    const state = await this.oauth.generateState(userId);
    
    // Build Stripe OAuth URL with required scopes
    const params = new URLSearchParams({
      response_type: 'code',
      client_id: process.env.STRIPE_CLIENT_ID,
      scope: 'read_write',
      redirect_uri: `${process.env.APP_URL}/stripe/callback`,
      state,
      // FinFlow specific parameters
      'stripe_user[business_type]': 'company',
      'stripe_user[country]': 'US'
    });
    
    return `https://connect.stripe.com/oauth/authorize?${params}`;
  }
  
  async handleCallback(code: string, state: string): Promise<StripeAccount> {
    // Verify state to prevent CSRF
    const userId = await this.oauth.verifyState(state);
    if (!userId) {
      throw new SecurityError('Invalid OAuth state');
    }
    
    // Exchange code for access token
    const response = await this.stripe.oauth.token({
      grant_type: 'authorization_code',
      code
    });
    
    // Store encrypted tokens
    await this.storeTokens(userId, response);
    
    // Fetch and store account details
    const account = await this.stripe.accounts.retrieve(
      response.stripe_user_id
    );
    
    return this.saveStripeAccount(userId, account);
  }
}

Case Study 2: The Legacy Rescue

Modernizing a 15-Year-Old Medical System

The Situation

Dr. Sarah Chen inherited IT responsibilities for a medical clinic using a 15-year-old patient management system. Written in PHP 5.2, using a custom framework, with 300,000 lines of code and zero documentation. The system couldn't be replaced—it contained irreplaceable medical history for 50,000 patients.

The Challenge

  • No original developers available
  • No documentation
  • Mixed encoding (Latin-1 and UTF-8)
  • Custom database abstraction layer
  • Hardcoded business logic
  • HIPAA compliance requirements
  • System must stay operational during updates

The Claude Code Exploration

$ claude --add-dir ./legacy_system --model claude-3-opus Dr. Chen: This system is a mystery to everyone. Can you help me understand what we're dealing with? Claude: I'll analyze this legacy system comprehensively. This will take a few moments as I map out the architecture... [Analysis complete] This is a fascinating piece of medical software history. Here's what I've discovered: ARCHITECTURE: - Custom MVC framework predating modern PHP frameworks - Database: MySQL 5.1 with 247 tables - Authentication: Custom session management - File structure: Partially follows PEAR conventions KEY FINDINGS: 1. Patient data is well-structured despite the age 2. Encryption for sensitive data (using mcrypt - deprecated!) 3. Audit logging system is comprehensive 4. Complex billing integration with clearinghouses IMMEDIATE CONCERNS: - 47 SQL injection vulnerabilities - Deprecated PHP functions throughout - Passwords using MD5 hashing - No CSRF protection - XSS vulnerabilities in patient notes Let me create a modernization plan that maintains operations...

The Modernization Strategy

Claude Code developed a careful, phased approach:

Phase 1: Security Hardening (Week 1-2)

Claude Code created a security wrapper to retrofit existing code with minimal changes.

Phase 2: Gradual PHP Upgrade (Week 3-6)

Claude Code created compatibility layers for PHP 5.2 → 7.4 migration.

Phase 3: API Layer Introduction (Week 7-10)

New REST API wrapping legacy code for gradual modernization.

Phase 4: Frontend Modernization (Week 11-16)

Claude Code created a React frontend that consumed the new API.

The Results

  • Security: All critical vulnerabilities patched
  • Performance: 3x faster page loads
  • Reliability: 99.9% uptime during migration
  • Compliance: Passed HIPAA audit
  • Developer Experience: New developers onboarded in days, not months
  • Future-Proof: Ready for continued modernization

Dr. Chen's Reflection

"Claude Code turned an impossible task into a methodical journey. It understood code written before it existed, found patterns in chaos, and helped us modernize without disruption. Most importantly, it respected the system's history while securing its future."
